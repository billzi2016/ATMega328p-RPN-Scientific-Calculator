# 基于 ATmega328P 的 RPN 科学计算器 PRD

## 1. 项目概述

### 1.1 项目名称
ATmega328P RPN Scientific Calculator

### 1.2 项目目标
基于 Arduino 生态与 ATmega328P 单片机，开发一套可落地、可维护、具备工业化组织方式的 RPN 科学计算器方案。设备应具备完整的数值输入、RPN 栈操作、科学函数、历史记录保存、文件化归档、基础模式切换与异常处理能力。

### 1.3 项目定位
- 面向嵌入式硬件实作，而不是桌面软件演示
- 面向长期维护，代码结构按 Arduino 工程规范分层
- 面向扩展，后续可追加程序员模式、配置持久化、自检、蜂鸣器、串口调试开关等能力

## 2. 范围定义

### 2.1 本期必须实现
- 使用 `ATmega328P` 作为主控
- 使用 `LCD2004` 显示
- 使用 `8x8` 键盘矩阵作为主输入设备
- 使用 `74HC595` 进行键盘行扫描输出
- 使用 `74HC165` 进行键盘列采样输入
- 避开 `TX/RX` 引脚
- 使用 `TF` 卡 `SPI` 存储历史算式与结果
- 历史文件采用 `1.txt / 2.txt / 3.txt ...` 顺延命名
- 采用 `RPN` 交互方式
- 给出工业项目化的软件分层、异常策略、文件树建议

### 2.2 本期建议实现
- `SHIFT` / `ALPHA` 双层按键逻辑
- 科学函数与角度模式切换
- 历史浏览、结果回填、文件查看
- 电源上电自检
- SD 卡异常提示
- 键盘去抖、长按、重复输入控制

### 2.3 本期不强制实现
- RTC 时间戳
- USB 数据导出工具
- 图形显示
- CAS 符号计算
- 可编程脚本语言

## 3. 硬件方案

### 3.1 主控
- 推荐：`Arduino Nano / Uno` 兼容板，核心为 `ATmega328P`
- 工作电压：`5V`
- 资源约束：
  - Flash：`32KB`
  - SRAM：`2KB`
  - EEPROM：`1KB`

### 3.2 显示模块
- 模块：`LCD2004`
- 接口：按 `4-bit` 并口模式设计
- 说明：
  - 标准 `LCD2004` 的 `4-bit` 并口通常需要 `6` 根控制/数据线：`RS`、`E`、`D4~D7`
  - 如果后续硬件要求被严格定义为“只能 4 根信号线”，则需要额外扩展器，不建议在本期强行压缩
  - 本 PRD 以“可落地优先”为原则，采用标准 `4-bit` 模式

### 3.3 键盘输入
- 键盘：`8x8` 矩阵，共 `64` 个物理按键位
- 行驱动：`74HC595`
- 列采样：`74HC165`
- 扫描方式：
  - 逐行激活
  - 逐列读取
  - 软件去抖
- 硬件约束：
  - `74HC165` 的列输入不能悬空
  - 列线应统一增加 `10k` 上拉电阻网络
  - 推荐采用“行低有效、列默认高、按下读低”的扫描逻辑
- 设计目标：
  - 尽量利用全部 `64` 个物理键位
  - 通过 `SHIFT / ALPHA / MODE` 扩展二级功能

### 3.4 存储
- 模块：`TF / MicroSD` SPI 模块
- 用途：
  - 保存本次开机周期内的全部计算记录
  - 提供历史浏览与回读

## 4. 核心需求

### 4.1 计算模型
- 使用 `RPN` 四级或多级栈模型
- 至少支持：
  - `X / Y / Z / T` 栈寄存器
  - `ENTER`
  - `DROP`
  - `DUP`
  - `SWAP`
  - `ROLL UP`
  - `ROLL DOWN`

### 4.2 基础运算
- `+`
- `-`
- `*`
- `/`
- `%`
- `+/-`
- 小数点输入
- 指数输入 `EXP`
- 括号输入用于表达式回显或代数兼容显示

### 4.3 科学计算
- 三角函数：`sin / cos / tan`
- 反三角函数：`asin / acos / atan`
- 对数函数：`ln / log`
- 幂函数：`x^2 / x^y`
- 开方：`sqrt`
- 倒数：`1/x`
- 常数：`π / e`
- 工程计数：`ENG`
- 角度模式：`DEG / RAD`

### 4.4 存储与历史
- 记录内容：
  - 输入算式或等价按键序列
  - 输出结果
  - 步号
  - 会话编号
- 文件命名规则：
  - 根目录或固定日志目录下保存 `1.txt`
  - 下一次新会话保存 `2.txt`
  - 依次类推
- 推荐策略：
  - 开机后先扫描现有最大编号
  - 当本次开机第一次成功完成计算后，创建 `N+1.txt`
  - 本次开机的后续计算均追加到同一个文件中
- 说明：
  - 不建议“每算一次就新建一个 txt”，否则文件数增长过快，不利于 FAT 文件系统长期维护

### 4.5 用户界面
- 主界面显示：
  - 当前输入
  - 栈顶结果
  - 角度模式
  - 存储状态
  - SD 卡状态
- 历史界面显示：
  - 当前会话编号
  - 当前记录编号
  - 算式摘要
  - 结果摘要
- 异常提示：
  - `SD ERR`
  - `DIV0`
  - `OVERFLOW`
  - `STACK ERR`
  - `FILE ERR`

## 5. 8x8 键位利用方案

### 5.1 结论
`8x8` 的 `64` 个物理键位可以全部利用，而且仍然建议增加 `SHIFT` 与 `ALPHA` 形成双层逻辑，以便在不牺牲主操作效率的前提下覆盖更完整功能。

详细逐键定义见 [KEYBOARD.md](./KEYBOARD.md)。

### 5.2 推荐主层键位分配

| 行\列 | C1 | C2 | C3 | C4 | C5 | C6 | C7 | C8 |
|---|---|---|---|---|---|---|---|---|
| R1 | ON | OFF | MENU | MODE | HIST | FILE | UP | DEL |
| R2 | SHIFT | ALPHA | LEFT | OK | RIGHT | ( | ) | BACK |
| R3 | 7 | 8 | 9 | / | sin | cos | tan | π |
| R4 | 4 | 5 | 6 | * | ln | log | x^y | e |
| R5 | 1 | 2 | 3 | - | sqrt | x^2 | 1/x | % |
| R6 | 0 | . | +/- | + | EXP | ENG | Ans | = |
| R7 | ENTER | SWAP | DROP | DUP | ROLL↑ | ROLL↓ | STO | RCL |
| R8 | MC | MR | M+ | M- | DEG | RAD | BIN | HEX |

### 5.3 副层功能建议
- `SHIFT + sin/cos/tan`：`asin / acos / atan`
- `SHIFT + ln/log`：`exp / 10^x`
- `SHIFT + DEG/RAD`：`GRAD / DMS`
- `SHIFT + BIN/HEX`：`OCT / DEC`
- `SHIFT + STO/RCL`：保存到指定槽 / 从指定槽回读
- `ALPHA`：在程序员模式或文件命名模式下输入 `A-F`

### 5.4 键位设计原则
- 高频输入放中部区域，减少手指移动
- 栈操作集中成组，便于 RPN 连续操作
- 文件、历史、模式、菜单放在上层导航区
- 程序员模式相关功能不抢占数值区
- 每个按键的主功能、副功能、使用语义统一维护在 [KEYBOARD.md](./KEYBOARD.md)

## 6. 工业项目化要求

### 6.1 软件架构
建议分为以下模块：
- `App`：主状态机、界面切换、模式控制
- `Keyboard`：矩阵扫描、去抖、长按判定、键值映射
- `Display`：LCD2004 驱动与页面渲染
- `CalculatorCore`：RPN 栈、运算分发、错误码
- `Storage`：SD 卡初始化、文件编号、日志追加、读取历史
- `Config`：角度模式、显示配置、开机选项
- `Diagnostics`：上电自检、错误上报、调试信息

### 6.2 可靠性要求
- 上电后执行硬件自检
- SD 卡初始化失败时计算器仍可工作，但历史存储功能降级
- 所有写文件操作需检查返回值并及时 `flush/close`
- 运算错误不得导致死机
- 栈溢出、除零、域错误必须统一错误码处理
- 键盘扫描周期固定，避免界面阻塞输入

### 6.3 可维护性要求
- `ino` 只保留初始化与主循环入口
- 业务逻辑全部拆到 `.cpp / .h`
- 注释必须使用中文，重点说明“为什么这样做”
- 每个源文件开头必须有文件头注释，说明文件职责、设计意图、主要依赖关系
- 每个对外函数和关键内部函数必须有函数注释，说明输入、输出、返回值、边界条件、异常情况
- 关键状态机、扫描流程、存储流程、错误处理流程必须有行内注释或段注释
- 重要常量、引脚定义、缓冲区长度、时间参数、错误码必须附中文说明
- 常量、引脚、菜单项、错误码统一集中定义
- 禁止把全部逻辑堆进单个 `.ino`

### 6.4 扩展性要求
- 预留 `D8 / D9` 作为蜂鸣器、卡检测或状态灯接口
- 预留模式扩展：
  - 科学模式
  - 程序员模式
  - 历史浏览模式
  - 维护/测试模式

## 7. Arduino 工程组织规范

本节是工程规范要求，当前阶段写入文档，不要求立即在仓库中创建对应代码目录。

### 7.1 推荐目录组织
```text
ino/
└── calculator/
    ├── calculator.ino
    ├── src/
    │   ├── app.cpp
    │   ├── calculator_core.cpp
    │   ├── display_lcd2004.cpp
    │   ├── keyboard_matrix.cpp
    │   ├── storage_sd.cpp
    │   └── diagnostics.cpp
    ├── include/
    │   ├── app.h
    │   ├── calculator_core.h
    │   ├── display_lcd2004.h
    │   ├── keyboard_matrix.h
    │   ├── storage_sd.h
    │   ├── diagnostics.h
    │   ├── pins.h
    │   ├── config.h
    │   └── types.h
    ├── lib/
    └── README.md
```

### 7.2 代码规范
- `calculator.ino`
  - 只放 `setup()` 与 `loop()`
  - 不直接写复杂业务逻辑
- `pins.h`
  - 集中定义全部 IO
- `config.h`
  - 集中定义缓冲区长度、栈深、文件路径、显示参数
- 模块间通过头文件暴露最小接口
- 所有 `.ino / .cpp / .h` 必须使用中文注释
- 每个文件开头必须写明本文件设计意图与职责边界
- 每个函数前必须写明用途、参数、返回值、调用时机
- 关键代码段需要有中文行注释或段注释，避免只看代码难以理解设计原因

## 8. 历史存储文件格式

### 8.1 目录建议
- `/LOGS/1.txt`
- `/LOGS/2.txt`
- `/LOGS/3.txt`

### 8.2 单条记录建议格式
```text
STEP=0001 | EXPR=12 ENTER 3 + | RESULT=15
STEP=0002 | EXPR=15 2 /       | RESULT=7.5
```

### 8.3 文件头建议
```text
SESSION=0001
MODE=RPN
ANGLE=DEG
SD=OK
----
```

## 9. 非功能需求

### 9.1 性能
- 单次按键响应时间：目标 `< 50ms`
- 普通算术计算显示刷新：目标 `< 100ms`

### 9.2 可用性
- 开机可直接进入计算界面
- 未插 SD 卡时可正常计算
- 异常时用户能看懂提示

### 9.3 可测试性
- 键盘扫描模块可独立测试
- 计算核心可脱离 LCD 做串口或单元测试
- 文件编号与日志格式可单独验证

## 10. 风险与约束项

### 10.1 SD 卡容量与寿命
- 高频写入会增加卡的磨损
- 建议按“每次开机会话一个文件”的策略降低碎片化

### 10.2 多键同时按下
- 无二极管矩阵下不建议依赖复杂多键组合
- `SHIFT` 类功能建议做成“先按后按”的时序逻辑，而不是完全同时按下

## 11. 验收标准
- 能完成四则运算与 RPN 栈操作
- 能显示科学函数结果
- 能正常扫描 `8x8` 键盘
- 不占用 `TX/RX`
- 能写入并顺延生成 `1.txt / 2.txt / 3.txt`
- 能在无 SD 卡时继续计算并提示降级
- 文档中明确了目录规范、接线规范、文件树规范
